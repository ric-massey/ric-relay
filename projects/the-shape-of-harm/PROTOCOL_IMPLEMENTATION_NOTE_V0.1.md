# Protocol implementation note 0.1 — reporting-window incompatibility

**Date:** 23 July 2026  
**Linked protocol:** PSILOCYBIN_SAFETY_REVIEW_PROTOCOL_V0.1.md  
**Status:** transparent implementation note; the primary estimand has not been changed.

Pilot extraction found that the two direct trials do not report every prespecified outcome in exact 0–24-hour and 0–7-day participant-level windows. The review will therefore:

1. Keep the original windows as primary.
2. Mark an outcome **not estimable** when those windows cannot be reconstructed.
3. Present nearby windows (for example, day 0, through day 9, or complete follow-up) only as labeled study-level context.
4. Never pool nearby windows as though they were identical.
5. Contact investigators for exact-window counts before a final review.

This note was created after inspecting reports, so it is not described as a fully prospective amendment. It narrows interpretation rather than changing eligibility or creating a favorable analysis.
