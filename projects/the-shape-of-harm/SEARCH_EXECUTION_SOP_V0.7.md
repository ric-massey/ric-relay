# Search Execution SOP v0.7

## Before any search

- Information specialist signs the PRESS-style review form.
- Final strategy is copied verbatim into `search-strategies-v0.6.csv` through a versioned amendment if changed.
- Registration is submitted and time-stamped.
- Searcher identity, platform, institutional access, and execution date are assigned.

## For every database or registry

Capture: exact platform; coverage; full query; filters; date and local time; raw result count; export format; exported filename; SHA-256 hash; and any interface warning or truncation. Export before screening. Never hand-edit the raw export.

## Public-source horizon scan versus independent search

The 13-record corpus in v0.7 is an author horizon scan used to test infrastructure. It is not the registered search result set and cannot populate the PRISMA database count. Independent searches start with new raw exports.

## Access failures

Licensed-source unavailability is documented, not silently omitted. Record the institution contacted, date, outcome, and whether an alternative database was used. A reduced-source review must be labeled and justified before screening.

## Updates

Rerun every source before final analysis. New records receive the same dual-screening workflow. Preserve both original and update PRISMA counts.
